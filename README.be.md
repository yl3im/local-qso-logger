Прыватны журнал QSO для радыёаматараў, які цалкам працуе ў вашым браўзеры. Без уліковага запісу, без сервера, без сачэння, без аналітыкі — вашы журналы захоўваюцца толькі ў `localStorage` браўзера і ніколі не пакідаюць вашу прыладу.

Аўтар: [YL3IM](https://www.qrz.com/db/YL3IM). Сайт праекта: [qso.ham.lv](https://qso.ham.lv).

## Магчымасці

- Некалькі журналаў, кожны са сваім спісам QSO.
- Дзеянні з журналам: стварэнне, перайменаванне, выдаленне, імпарт з ADIF, экспарт у ADIF (`.adi`).
- Палі QSO: пазыўны, дата UTC, час UTC, дыяпазон, від, RST перададзены, RST прыняты.
- Рэдагаванне і выдаленне любога QSO (з пацвярджэннем пры выдаленні).
- Разумныя значэнні па змаўчанні: сённяшнія дата/час UTC ужо запоўненыя, RST па змаўчанні залежаць ад віду (59 для галасавых, 599 для CW/лічбавых), дыяпазон і від застаюцца паміж паслядоўнымі QSO.
- Жывы індыкатар дублюючага пазыўнога (інфарматыўны — дублікаты дазволены).
- Калонка з сцягам краіны, вылучаная з прэфікса пазыўнога (пакрывае ≥99 % звычайных радыёаматарскіх прэфіксаў, уключаючы пераносныя пазыўныя кшталту `9A/M0NCG`).
- Адлюстраванне даты ў таблiцы QSO з улiкам лакалi; ISO-захоўванне і ADIF-выхад застаюцца без змен.
- Дзённая/начная тэмы (дзень па змаўчанні; пераключальнік у загалоўку).
- Зручны для мабільных адаптыўны макет з кнопкамі зручнага для дотыку памеру.
- Працуе цалкам афлайн — ніякіх сеткавых запытаў.
- Усталёўваецца як PWA (Дадаць на галоўны экран / Усталяваць праграму), калі размешчаны па HTTPS.
- Інтэрфэйс даступны на **28 мовах** (англійская плюс 22 лацінскія, 5 кірыліцы і грэчаская); селектар з сцягамі ў загалоўку.

## Пачатак працы

Проста адкрыйце `index.html` у сучасным браўзеры. Без зборкі, без усталявання, без сервера.

Каб размясціць, пакладзіце статычныя файлы (`index.html`, `style.css`, `app.js`, `favicon.svg`, `manifest.webmanifest`, `service-worker.js` і каталог `i18n/` з 28 файламі перакладаў) на любы статычны хост (GitHub Pages, Netlify, ваш уласны вэб-сервер). Будзе працаваць і праз `file://` — рэгістрацыя service worker аўтаматычна прапускаецца для пратакола `file:`, таму прамое адкрыццё `index.html` з дыска працуе чыста.

Пры абслугоўванні праз HTTPS прыкладанне становіцца ўсталёўваным як PWA (праз меню браўзера *Усталяваць праграму* / *Дадаць на галоўны экран*) і працуе афлайн пасля першага наведвання дзякуючы cache-first service worker, які папярэдне кэшуе кожны статычны файл (UI + усе пераклады).

Пры першым наведванні аўтаматычна ствараецца журнал па змаўчанні, таму вы можаце адразу пачаць запісваць.

## Усталяванне як PWA на мабільным

Калі прыкладанне абслугоўваецца праз HTTPS (напрыклад, GitHub Pages), вы можаце ўсталяваць яго на галоўны экран тэлефона, каб яно працавала ў поўнаэкранным рэжыме без браўзернай абалонкі. Пасля першага запуску service worker кэшуе ўсё, таму далейшыя запускі працуюць цалкам афлайн.

### iOS (толькі Safari)

На iOS толькі Safari можа ўсталёўваць PWA — староннія браўзеры не могуць.

2. Націсніце кнопку **Падзяліцца**.
3. Выберыце **На «Дамашні» экран**, потым **Дадаць**.

## Журналы

- **Стварыць:** увядзіце назву ў поле *Назва журнала* і адпраўце. Калі пакінуць назву пустой, па змаўчанні будзе `Log YYYY-MM-DD HH:MM UTC`.
- **Пераключыць:** націсніце на любы журнал у бакавой панэлі.
- **Перайменаваць:** націсніце *Перайменаваць* у загалоўку журнала. Enter захоўвае, Escape скасоўвае.
- **Выдаліць:** націсніце *Выдаліць журнал*. Будзе прапанавана пацвердзіць. Калі выдалiць апошнi журнал, аўтаматычна створыцца новы.

## QSO

- Запоўніце форму і націсніце **Запісаць QSO**.
- Пазыўны аўтаматычна пераводзіцца ў верхні рэгістр пры наборы.
- Дата і час запаўняюцца *цяпер* у UTC і скідаюцца пасля кожнага запісанага QSO; вы можаце ўвесці любое значэнне.
- Дыяпазон і від захоўваюцца паміж QSO у адной сесіі, таму не трэба выбіраць зноў для кожнага кантакту.
- RST перададзены / RST прыняты, калі пакінуты пустымі, па змаўчанні становяцца **59** для галасавых відаў (SSB/FM/AM/DIGITALVOICE) і **599** для CW і лічбавых відаў (CW/FT8/FT4/RTTY/PSK31/JT65).
- Чып *Дублікат у гэтым журнале* з'яўляецца пад полем пазыўнога, калі пазыўны ўжо існуе ў бягучым журнале. Дублікаты *не* блакіруюцца.
- **Рэдагаваць QSO** кнопкай *Змен.* у радку. Форма пераключаецца ў рэжым *Абнавіць QSO*, радок падсвятляецца, з'яўляецца кнопка *Скасаваць*. Пераключэнне журнала або выдаленне журнала аўтаматычна скасоўвае рэдагаванне.
- **Выдаліць QSO** кнопкай *Выдал.* у радку (запытвае пацверджанне).

## Імпарт і экспарт ADIF

- **Экспарт**: націсніце *Экспарт .adi* у загалоўку журнала. Спампоўваецца файл з `ADIF_VER 3.1.4` і `PROGRAMID local-qso` у загалоўку. Кожны запіс адлюстроўвае `CALL`, `QSO_DATE`, `TIME_ON`, `BAND`, `MODE`, `RST_SENT`, `RST_RCVD`.
- **Імпарт**: націсніце *Імпарт файла .adi* пад формай стварэння журнала і выберыце файл `.adi`/`.adif`. Ствараецца новы журнал з назвай `Імпарт YYYY-MM-DD HH:MM UTC`. Імпарт ніколі не зліваецца з існуючым журналам.
- Даўжыня поля інтэрпрэтуецца як колькасць знакаў, што працуе для ASCII ADIF (усіх стандартных палёў QSO). Шматбайтавы змест у неістотных тэкставых палях можа парсіцца дзіўна.

## Прыватнасць і даныя

- Усе даныя захоўваюцца ў `localStorage` браўзера пад ключом `local-qso:v1`.
- Нічога нікуды не перадаецца. Без бэкенду, без API-выклікаў, без тэлеметрыі, без аналітыкі.
- Ачыстка даных сайта, выкарыстанне прыватнага/інкогніта рэжыму або іншага браўзера/прылады азначае пусты журнал — выкарыстоўвайце *Экспарт .adi* для рэзервовай копіі.

## Тэмы

Пераключальнік тэм у загалоўку пераключае паміж днём (па змаўчанні) і ноччу. Налада захоўваецца разам з журналамі і шануецца пры наступным наведванні. Натыўныя селектары даты/часу прытрымліваюцца тэмы праз `color-scheme`.

## Тэхнічныя нататкі

- Аднастаронкавае прыкладанне, чысты HTML + CSS + JavaScript. Без фрэймворкаў, без зборкі, без залежнасцей.
- Зыходныя файлы:
  - `index.html` — разметка і мета-тэгі.
  - `style.css` — тэмы і макет (зменныя дзень/ноч, медыя-запыты для мабільных).
  - `app.js` — стан, persistance, рэндэрынг, ADIF парсэр/сэрыялізатар, пошук прэфікса пазыўнога → краіна.
  - `favicon.svg` — убудаваны SVG favicon.
  - `manifest.webmanifest` — Web App Manifest (назва, колер тэмы, scope, значок), каб прыкладанне можна было ўсталяваць як PWA на мабільным і дэсктопе.
  - `service-worker.js` — cache-first service worker, які пры ўсталяванні папярэдне кэшуе кожны статычны файл, пры актывацыі выдаляе старыя кэшы і падтрымлівае прыкладанне цалкам афлайн пасля першага наведвання. Рэгістрацыя аўтаматычна прапускаецца для пратакола `file://`, таму прамое адкрыццё `index.html` з дыска застаецца чыстым.
  - `i18n/<lang>.js` — адзін файл перакладу на кожную падтрымліваемую мову (28 усяго). Кожны — маленькі IIFE, які прысвойвае `window.I18N[<lang>]` плоскую карту ключ→радок. `t()` і `applyLanguage()` у `app.js` апрацоўваюць пошукі (з англійскім fallback) і праходзяць па DOM, абнаўляючы кожны `[data-i18n*]` элемент.
- Пратэставана ў апошніх Chromium, Firefox і Safari (дэсктоп + iOS).
