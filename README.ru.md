QSO-журнал для радиолюбителей, уважающий приватность, который полностью работает в вашем браузере. Без аккаунта, без сервера, без слежки, без аналитики — ваши журналы хранятся только в `localStorage` браузера и никогда не покидают ваше устройство.

## Возможности

- Несколько журналов, каждый со своим списком QSO.
- Действия с журналом: создание, переименование, удаление, импорт из ADIF, экспорт в ADIF (`.adi`).
- Поля QSO: позывной, дата UTC, время UTC, диапазон, вид, RST передан, RST принят.
- Редактирование и удаление любого QSO (с подтверждением при удалении).
- Разумные значения по умолчанию: сегодняшние дата/время UTC уже заполнены, RST по умолчанию зависят от вида (59 для голосовых, 599 для CW/цифровых), диапазон и вид остаются между последовательными QSO.
- Живой индикатор дублирующегося позывного (информационный — дубликаты разрешены).
- Колонка с флагом страны, выводится из префикса позывного (покрывает ≥99 % обычных любительских префиксов, включая портативные позывные вроде `9A/M0NCG`).
- Отображение даты в таблице QSO в соответствии с локалью; ISO-хранение и ADIF-вывод остаются неизменными.
- Дневная/ночная темы (день по умолчанию; переключатель в шапке).
- Удобный для мобильных устройств адаптивный макет с кнопками удобного для касания размера.
- Полностью работает офлайн — никаких сетевых запросов.
- Устанавливается как PWA (Добавить на главный экран / Установить приложение), если размещён по HTTPS.
- Интерфейс доступен на **28 языках** (английский плюс 22 на латинице, 5 на кириллице и греческий); селектор с флагами в шапке.

## Начало работы

Просто откройте `index.html` в современном браузере. Без сборки, без установки, без сервера.

Если хотите разместить, поместите статические файлы (`index.html`, `style.css`, `app.js`, `favicon.svg`, `manifest.webmanifest`, `service-worker.js` и каталог `i18n/` с 28 файлами переводов) на любой статический хост (GitHub Pages, Netlify, ваш собственный веб-сервер). Будет работать и через `file://` — регистрация service worker автоматически пропускается для протокола `file:`, поэтому прямое открытие `index.html` с диска работает чисто.

При обслуживании через HTTPS приложение становится устанавливаемым как PWA (через меню браузера *Установить приложение* / *Добавить на главный экран*) и работает офлайн после первого посещения благодаря cache-first service worker, который предварительно кэширует каждый статический файл (UI + все переводы).

При первом посещении автоматически создаётся журнал по умолчанию, чтобы вы могли сразу начать записывать.

## Установка как PWA на мобильном устройстве

Когда приложение обслуживается через HTTPS (например, GitHub Pages), вы можете установить его на главный экран телефона, чтобы оно работало в полноэкранном режиме без браузерной обёртки. После первого запуска service worker кэширует всё, поэтому последующие запуски работают полностью офлайн.

### iOS (только Safari)

На iOS только Safari может устанавливать PWA — сторонние браузеры не могут.

1. Откройте сайт в **Safari**.
2. Нажмите кнопку **Поделиться**.
3. Выберите **На экран «Домой»**, затем **Добавить**.

## Журналы

- **Создать:** введите имя в поле *Название журнала* и отправьте. Если оставить имя пустым, по умолчанию будет `Log YYYY-MM-DD HH:MM UTC`.
- **Переключить:** нажмите на любой журнал в боковой панели.
- **Переименовать:** нажмите *Переименовать* в шапке журнала. Enter сохраняет, Escape отменяет.
- **Удалить:** нажмите *Удалить журнал*. Будет предложено подтвердить. Если удалить последний журнал, автоматически создастся новый.

## QSO

- Заполните форму и нажмите **Записать QSO**.
- Позывной автоматически переводится в верхний регистр при наборе.
- Дата и время заполняются *сейчас* в UTC и сбрасываются после каждого записанного QSO; вы можете ввести любое значение.
- Диапазон и вид сохраняются между QSO в одной сессии, поэтому не нужно выбирать заново для каждого контакта.
- RST передан / RST принят, если оставлены пустыми, по умолчанию **59** для голосовых видов (SSB/FM/AM/DIGITALVOICE) и **599** для CW и цифровых видов (CW/FT8/FT4/RTTY/PSK31/JT65).
- Чип *Дубликат в журнале* появляется под полем позывного, если позывной уже существует в текущем журнале. Дубликаты *не* блокируются.
- **Редактировать QSO** кнопкой *Изм.* в строке. Форма переключается в режим *Обновить QSO*, строка подсвечивается, появляется кнопка *Отмена*. Переключение журнала или удаление журнала автоматически отменяет редактирование.
- **Удалить QSO** кнопкой *Удал.* в строке (запрашивает подтверждение).

## Импорт и экспорт ADIF

- **Экспорт**: нажмите *Экспорт .adi* в шапке журнала. Скачивается файл с `ADIF_VER 3.1.4` и `PROGRAMID local-qso` в заголовке. Каждая запись сопоставляет `CALL`, `QSO_DATE`, `TIME_ON`, `BAND`, `MODE`, `RST_SENT`, `RST_RCVD`.
- **Импорт**: нажмите *Импорт файла .adi* под формой создания журнала и выберите `.adi`/`.adif` файл. Создаётся новый журнал с именем `Импорт YYYY-MM-DD HH:MM UTC`. Импорт никогда не сливается с существующим журналом.
- Длина поля интерпретируется как число символов, что работает для ASCII ADIF (всех стандартных полей QSO). Многобайтное содержимое в несущественных текстовых полях может парситься странно.

## Приватность и данные

- Все данные хранятся в `localStorage` браузера под ключом `local-qso:v1`.
- Ничего никуда не передаётся. Нет бэкенда, нет API-вызовов, нет телеметрии, нет аналитики.
- Очистка данных сайта, использование приватного/инкогнито режима, или использование другого браузера/устройства означает пустой журнал — используйте *Экспорт .adi* для резервного копирования.

Универсальные технические метки остаются в канонической форме во всех языках: названия диапазонов (`20m`, `70cm`, …), коды видов ADIF (`SSB`, `FT8`, `CW`, …), `QSO`, `RST`, `UTC` и ISO-коды стран.

## Темы

Переключатель тем в шапке переключает между днём (по умолчанию) и ночью. Предпочтение сохраняется вместе с вашими журналами и учитывается при следующем посещении. Нативные выбиратели даты/времени следуют теме через `color-scheme`.

## Технические заметки

- Одностраничное приложение, чистый HTML + CSS + JavaScript. Никаких фреймворков, никакой сборки, никаких зависимостей.
- Исходные файлы:
  - `index.html` — разметка и мета-теги.
  - `style.css` — темы и макет (переменные день/ночь, медиа-запросы для мобильных).
  - `app.js` — состояние, персистентность, рендеринг, ADIF парсер/сериализатор, поиск префикс позывного → страна.
  - `favicon.svg` — встроенный SVG favicon.
  - `manifest.webmanifest` — Web App Manifest (имя, цвет темы, scope, иконка), чтобы приложение можно было устанавливать как PWA на мобильном и десктопе.
  - `service-worker.js` — cache-first service worker, который при установке предварительно кэширует каждый статический файл, при активации удаляет старые кэши и поддерживает приложение полностью офлайн после первого посещения. Регистрация автоматически пропускается для протокола `file://`, поэтому прямое открытие `index.html` с диска остаётся чистым.
  - `i18n/<lang>.js` — один файл перевода на каждый поддерживаемый язык (всего 28). Каждый — это маленькая IIFE, которая присваивает `window.I18N[<lang>]` плоскую карту ключ→строка. `t()` и `applyLanguage()` в `app.js` обрабатывают поиск (с откатом на английский) и проходят по DOM, обновляя каждый элемент `[data-i18n*]`.
- Протестировано в последних Chromium, Firefox и Safari (десктоп + iOS).
