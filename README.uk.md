Журнал QSO для радіоаматорів, що шанує приватність і повністю працює у вашому браузері. Без облікового запису, без сервера, без відстеження, без аналітики — ваші журнали зберігаються лише в `localStorage` браузера і ніколи не залишають ваш пристрій.

Автор: [YL3IM](https://www.qrz.com/db/YL3IM). Сайт проєкту: [qso.ham.lv](https://qso.ham.lv).

## Можливості

- Декілька журналів, кожен зі своїм списком QSO.
- Дії з журналом: створення, перейменування, видалення, імпорт з ADIF, експорт в ADIF (`.adi`).
- Поля QSO: позивний, дата UTC, час UTC, діапазон, вид, RST передано, RST прийнято.
- Редагування і видалення будь-якого QSO (з підтвердженням при видаленні).
- Розумні значення за замовчуванням: сьогоднішні дата/час UTC вже заповнені, RST за замовчуванням залежать від виду (59 для голосових, 599 для CW/цифрових), діапазон і вид залишаються між послідовними QSO.
- Живий індикатор дублюючого позивного (інформативний — дублікати дозволені).
- Колонка з прапором країни, виведена з префіксу позивного (покриває ≥99 % звичайних радіоаматорських префіксів, включно з портативними позивними на кшталт `9A/M0NCG`).
- Локалі-обізнане відображення дати в таблиці QSO; ISO-зберігання та ADIF-вихід залишаються без змін.
- Денна/нічна теми (день за замовчуванням; перемикач у заголовку).
- Зручний для мобільних адаптивний макет з кнопками зручного для дотику розміру.
- Працює повністю офлайн — жодних мережевих запитів.
- Встановлюється як PWA (Додати на головний екран / Встановити застосунок), якщо розміщено по HTTPS.
- Інтерфейс доступний у **28 мовах** (англійська плюс 22 латинські, 5 кириличних і грецька); селектор з прапорами в заголовку.

## Початок роботи

Просто відкрийте `index.html` у сучасному браузері. Без збірки, без встановлення, без сервера.

Щоб розмістити, помістіть статичні файли (`index.html`, `style.css`, `app.js`, `favicon.svg`, `manifest.webmanifest`, `service-worker.js` і каталог `i18n/` з 28 файлами перекладів) на будь-який статичний хост (GitHub Pages, Netlify, ваш власний веб-сервер). Буде працювати і через `file://` — реєстрація service worker автоматично пропускається для протоколу `file:`, тому пряме відкриття `index.html` з диска працює чисто.

При обслуговуванні через HTTPS застосунок стає встановлюваним як PWA (через меню браузера *Встановити застосунок* / *Додати на головний екран*) і працює офлайн після першого відвідування завдяки cache-first service worker, який попередньо кешує кожен статичний файл (UI + усі переклади).

При першому відвідуванні автоматично створюється журнал за замовчуванням, тож ви можете одразу почати записувати.

## Встановлення як PWA на мобільному

Коли застосунок обслуговується через HTTPS (наприклад, GitHub Pages), ви можете встановити його на головний екран телефону, щоб він працював у повноекранному режимі без браузерної оболонки. Після першого запуску service worker кешує все, тому подальші запуски працюють повністю офлайн.

## Журнали

- **Створити:** введіть назву в поле *Назва журналу* і надішліть. Якщо залишити назву порожньою, за замовчуванням буде `Log YYYY-MM-DD HH:MM UTC`.
- **Переключити:** натисніть на будь-який журнал у бічній панелі.
- **Перейменувати:** натисніть *Перейменувати* у заголовку журналу. Enter зберігає, Escape скасовує.
- **Видалити:** натисніть *Видалити журнал*. Буде запропоновано підтвердити. Якщо видалити останній журнал, автоматично створиться новий.

## QSO

- Заповніть форму і натисніть **Записати QSO**.
- Позивний автоматично переводиться у верхній регістр під час набору.
- Дата і час заповнюються *зараз* в UTC і скидаються після кожного записаного QSO; ви можете ввести будь-яке значення.
- Діапазон і вид зберігаються між QSO в одній сесії, тож не потрібно вибирати знову для кожного контакту.
- RST передано / RST прийнято, якщо залишені порожніми, за замовчуванням стають **59** для голосових видів (SSB/FM/AM/DIGITALVOICE) і **599** для CW і цифрових видів (CW/FT8/FT4/RTTY/PSK31/JT65).
- Чип *Дублікат у цьому журналі* з'являється під полем позивного, якщо позивний вже існує в поточному журналі. Дублікати *не* блокуються.
- **Редагувати QSO** кнопкою *Змін.* у рядку. Форма перемикається в режим *Оновити QSO*, рядок підсвічується, з'являється кнопка *Скасувати*. Переключення журналу або видалення журналу автоматично скасовує редагування.
- **Видалити QSO** кнопкою *Видал.* у рядку (запитує підтвердження).

## Імпорт та експорт ADIF

- **Експорт**: натисніть *Експорт .adi* у заголовку журналу. Завантажується файл з `ADIF_VER 3.1.4` та `PROGRAMID local-qso` у заголовку. Кожен запис відображає `CALL`, `QSO_DATE`, `TIME_ON`, `BAND`, `MODE`, `RST_SENT`, `RST_RCVD`.
- **Імпорт**: натисніть *Імпорт файлу .adi* під формою створення журналу і виберіть файл `.adi`/`.adif`. Створюється новий журнал з назвою `Імпорт YYYY-MM-DD HH:MM UTC`. Імпорт ніколи не зливається з існуючим журналом.
- Довжина поля інтерпретується як кількість символів, що працює для ASCII ADIF (усіх стандартних полів QSO). Багатобайтовий вміст у неістотних текстових полях може парситися дивно.

## Приватність та дані

- Усі дані зберігаються у `localStorage` браузера під ключем `local-qso:v1`.
- Нічого нікуди не передається. Без бекенду, без API-викликів, без телеметрії, без аналітики.
- Очищення даних сайту, використання приватного/інкогніто режиму або іншого браузера/пристрою означає порожній журнал — використовуйте *Експорт .adi* для резервного копіювання.

## Теми

Перемикач теми в заголовку перемикає між днем (за замовчуванням) і ніччю. Налаштування зберігається разом з журналами і шанується при наступному відвідуванні. Нативні селектори дати/часу слідують темі через `color-scheme`.

## Технічні нотатки

- Односторінковий застосунок, чистий HTML + CSS + JavaScript. Без фреймворків, без збірки, без залежностей.
- Файли вихідного коду:
  - `index.html` — розмітка та мета-теги.
  - `style.css` — теми та макет (змінні день/ніч, медіа-запити для мобільних).
  - `app.js` — стан, persistance, рендеринг, ADIF парсер/серіалізатор, пошук префікса позивного → країна.
  - `favicon.svg` — вбудований SVG favicon.
  - `manifest.webmanifest` — Web App Manifest (назва, колір теми, scope, іконка), щоб застосунок можна було встановити як PWA на мобільному і десктопі.
  - `service-worker.js` — cache-first service worker, який при встановленні попередньо кешує кожен статичний файл, при активації видаляє старі кеші і підтримує застосунок повністю офлайн після першого відвідування. Реєстрація автоматично пропускається для протоколу `file://`, тож пряме відкриття `index.html` з диска залишається чистим.
  - `i18n/<lang>.js` — один файл перекладу на кожну підтримувану мову (28 загалом). Кожен — маленький IIFE, який присвоює `window.I18N[<lang>]` пласку карту ключ→рядок. `t()` і `applyLanguage()` в `app.js` обробляють пошук (з англійським fallback) і проходять по DOM, оновлюючи кожен `[data-i18n*]` елемент.
- Протестовано в останніх Chromium, Firefox та Safari (десктоп + iOS).
